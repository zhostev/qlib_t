# 实验列表页面优化计划

## 问题分析
1. 当前实验列表没有按创建时间排序，用户无法快速找到最新创建的实验
2. 只有卡片式一种显示样式，缺乏灵活性
3. 需要添加样式切换功能，支持卡片和列表两种样式

## 解决方案

### 1. 排序功能实现
- 在前端对实验列表按 `created_at` 字段进行倒序排序
- 在 `useEffect` 中获取数据后进行排序，或在渲染前进行排序
- 确保新创建的实验始终显示在列表前面

### 2. 样式切换功能实现
- 添加 `viewMode` 状态变量，支持 'card' 和 'list' 两种模式
- 添加切换按钮，允许用户在两种模式间切换
- 为列表模式设计新的 CSS 样式
- 根据 `viewMode` 动态应用不同的样式

### 3. 列表样式设计
- 设计简洁的列表样式，显示实验的关键信息
- 每行显示实验名称、状态、创建时间和操作按钮
- 保持与卡片样式的视觉一致性
- 支持响应式设计

## 代码修改点

### 1. Experiments.tsx
- 添加 `viewMode` 状态变量
- 添加排序逻辑，按 `created_at` 倒序
- 添加样式切换按钮
- 根据 `viewMode` 动态渲染不同样式的实验列表

### 2. CSS 样式
- 为列表模式添加新的 CSS 类和样式
- 确保两种样式可以无缝切换
- 保持样式的一致性和美观性

## 实现步骤

1. 在 `Experiments.tsx` 中添加排序逻辑
2. 添加 `viewMode` 状态和切换按钮
3. 实现列表样式的渲染逻辑
4. 添加列表样式的 CSS
5. 测试排序和样式切换功能

## 预期效果
- 实验列表默认按最后创建时间排序
- 用户可以在卡片和列表样式间切换
- 两种样式都能清晰展示实验信息
- 样式切换功能流畅，视觉效果一致
