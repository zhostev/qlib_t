# 远程调用训练任务联调测试计划

## 问题分析
1. 当前配置的训练服务器URL（http://144.0.23.148:8000）无法连接（Connection refused）
2. 用户需要将训练任务远程调用到ddns.hoo.ink服务器，但连接超时
3. 需要确保训练服务在ddns.hoo.ink服务器上正确运行并可访问

## 解决方案

### 1. 检查网络连接和服务器状态
- 测试ddns.hoo.ink服务器的基本连通性
- 检查服务器8000端口是否开放
- 确认训练服务是否在服务器上运行

### 2. 配置更新
- 更新配置文件，将TRAINING_SERVER_URL设置为ddns.hoo.ink
- 确保配置正确应用到系统中

### 3. 测试连接
- 测试从后端到ddns.hoo.ink服务器的API调用
- 验证训练任务能够正确启动和状态查询

### 4. 调试和优化
- 检查可能的网络防火墙问题
- 优化超时设置
- 添加更详细的日志记录

## 实施步骤

1. **检查网络连接**
   - 使用ping命令测试ddns.hoo.ink服务器连通性
   - 使用telnet或nc命令检查8000端口是否开放
   - 检查服务器上的训练服务运行状态

2. **更新配置**
   - 修改backend/app/config.py中的默认TRAINING_SERVER_URL
   - 或创建.env文件设置TRAINING_SERVER_URL环境变量

3. **测试API调用**
   - 使用curl命令测试ddns.hoo.ink服务器的API端点
   - 运行后端服务，测试训练任务启动功能
   - 检查日志，确认调用是否成功

4. **调试问题**
   - 如果连接失败，检查网络防火墙设置
   - 检查服务器上的训练服务日志
   - 优化超时设置和错误处理

## 预期结果
- 成功连接到ddns.hoo.ink服务器上的训练服务
- 训练任务能够正常启动和执行
- 能够正确查询训练任务状态和结果
- 前后端联调测试通过
